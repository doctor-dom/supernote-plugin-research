## Bugs

| ID | Status | Title | Notes |
|----|--------|-------|-------|
| B-004 | Open | Selected projects in settings isn't honored in the upcoming or today area or projects of supertask | Projects tab fixed: shows all projects (including empty ones), honors `enabledProjectIds` when set. Today/Upcoming tab filtering still open. |
| B-005 | Open | Renaming a note breaks task back-references | Both Todoist description (`Captured from: file.note p.N`) and task registry (`noteFile`) store the bare filename. Renaming the .note file breaks View Note navigation and This Page scan. **Approach:** reconciliation on plugin open -- if a registry noteFile is missing from disk, scan .note files for pages containing a supertask:// link matching the task ID (link elements survive renames). When found, update registry `noteFile` + patch Todoist task description via API. Short-circuit by checking same directory first. |
| B-007 | Done | Long press fires on any empty space (single-link fallback) | `preScanLinks` had a fallback: if only 1 supertask link on page and hit-test fails, use it anyway. This caused ANY long press ANYWHERE on a page with a link to open that task. Fix: removed the fallback entirely. If touch doesn't hit link bounds, long press is ignored. Users access tasks via toolbar/This Page if bounds are unreliable. |
| B-013 | Fixed | recognizeElements error 117 on lower page region (EMR/page size mismatch) | A5X (deviceType=3, fileMachineType=3) reports pageSize 1404x1872 but stroke EMR values reach maxX=20967 maxY=15725 (A5X2 digitizer range, normal max is 15819/11864). Passing reported pageSize to `recognizeElements` causes error 117 for strokes below ~y=1400 pixels. Content in upper page works because EMR values stay within normal range. Confirmed across 3 pages, fresh content, different positions. Native "Convert to Text" works on same content (doesn't use plugin API size param). Fix: detect EMR range from element maxX/maxY and pass A5X2 page size (1920x2560) when EMR values exceed normal range. |
| B-012 | Open | Phantom pen/multi-touch events cancel gesture after sleep | Digitizer reports phantom pen events (toolType=2) and instant PTR_DOWN[1] at identical coordinates to initial DOWN during finger holds. Consistently reproducible after device wake from sleep. Causes repeated gesture cancellation + 500ms cooldown, making gestures feel hung/delayed until transients clear. User observes gesture eventually works after waiting -- rules out genuine multi-touch (which would never resolve). Two phantom sources: (1) PEN events with no real pen contact (digitizer wake noise), (2) PTR_DOWN with same coords and sequential event number (touch controller calibration glitch). Potential fix: debounce PEN (require 2+ events before cancel), ignore PTR_DOWN at identical coords within ~50ms of initial DOWN. |
| B-014 | Open | Hardcoded A5X page size (1404x1872) may break Nomad gestures and OCR | Multiple locations fall back to A5X dimensions when `getPageSize()` fails or use A5X EMR constants as detection boundaries. On Nomad (deviceType=4, unknown page dimensions/EMR range), this could cause: (1) incorrect lasso bounds from wrong EMR-to-pixel scale factors, (2) `recognizeElements` error 117 if page size is wrong, (3) temp link positioned incorrectly. **Affected files:** `ocr.js:91` (hardcoded default 1404x1872), `Capture.tsx:46-59` (EMR constants assume only A5X/Manta ranges, scale factor calc at lines 183-184), `tempLinkNav.js:96-97` (hardcoded fallback). **Not affected:** `gestureDetector.js` (uses runtime page height, fail-closed). **To confirm:** need Nomad `getDeviceType()` + `getPageSize()` + element maxX/maxY from device logs. **Fix approach:** call `getDeviceType()` at startup, map to correct dimensions per device, remove hardcoded A5X fallbacks (fail gracefully or use device-specific map). |
